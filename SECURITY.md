# Security Policy

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you believe you've found a security vulnerability in Stats Umami, report it privately by email to
**info@umamiwp.com**. Include a description, steps to reproduce or a proof of concept, and the plugin
version affected.

We will acknowledge your report, investigate, and keep you updated on the fix. Once a fix is released,
we're happy to credit you (unless you prefer to remain anonymous).

## Supported versions

The latest released version receives security fixes. Please keep Stats Umami up to date.

## Scope note

Stats Umami is entirely client-side and stores no Umami credentials on your site. It injects the
official Umami tracker so your visitors' browsers send analytics directly to **your own** Umami
instance. Vulnerabilities in Umami itself should be reported to the [Umami project](https://umami.is).
